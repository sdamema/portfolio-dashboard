Da commenti grezzi a input puliti per pagine recensioni.

Questa pagina riassume il flusso operativo usato per raccogliere, normalizzare e validare commenti social prima di passarli alla pipeline di clustering, sentiment e scoring.

Flusso operativo

1. Target e fonti 2. Raccolta raw 3. Normalizzazione 4. Export reviews-ready

Controlli qualita

  • Conservazione degli export grezzi separata dagli output normalizzati.
  • URL, autore, piattaforma e testo commento mantenuti come campi tracciabili.
  • Esclusione di referral, promozioni personali e commenti senza fonte verificabile.
  • Source log per query, limiti di raccolta e note di validazione.

Schema normalizzato

I file puliti convergono in una struttura unica, pensata per alimentare clustering e sentiment senza riscrivere i commenti originali.

local_id, fornitore, commento, autore, platform, source, source_url, data, scraped_at, n, note

Output per la pipeline recensioni

  1. CSV raw archiviato per audit.
  2. CSV normalizzato con schema completo.
  3. CSV storico compatibile con reviews/data/raw.
  4. Report di validazione prima dell'uso editoriale.

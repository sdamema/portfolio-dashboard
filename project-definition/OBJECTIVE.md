# Dashboard Objective

Build a public, interactive SEO portfolio dashboard that helps recruiters and hiring managers understand how the work was done, not only what the final pages look like.

## Core Positioning

The dashboard presents Emanuele Sorgona as an SEO specialist who can connect:

- data collection and cleaning;
- keyword and content architecture;
- sentiment and review analysis;
- internal linking and page production;
- HTML/CSS/JS modules ready for WordPress;
- reporting and business-oriented SEO analysis.

## Experience Goal

The first screen should feel like a workbench: structured, measurable and interactive. A visitor should be able to click a cluster, understand the workflow, and open a real Italian page/module example.

## Current MVP

The MVP is static and GitHub Pages-ready:

- English dashboard UI.
- Italian demos preserved as original page outputs.
- Project cards open dedicated detail views with summaries, deep-dive accordions, key numbers, data sections, URL inventories and HTML demos.
- Main SEO clusters: reviews, offer sheets, branch map, review summary, provider ranking, NRJ2 analysis, scraping and secondary digital/content experiences.
- Review cluster now includes a processed GSC performance chart and URL inventory.
- No build step and no backend.

## Success Criteria

- The portfolio is live from a GitHub Pages URL.
- The dashboard loads on desktop and mobile.
- Every cluster has a clear process, evidence and live demo.
- Cluster performance data is shown only through sanitized/aggregated public files.
- Public files are safe to show externally.

# Portfolio Review Backlog

Working notes for reviewing the current public dashboard step by step before deciding which changes to implement.

Date: July 6, 2026.

## Review Lens

- Primary reader: recruiter or hiring manager for SEO, digital marketing, content marketing or marketing analytics roles.
- Secondary reader: technical/content stakeholder who may inspect workflow quality in more detail.
- Main goal: make the portfolio feel like a guided hiring argument, not only a complete archive of projects.
- Keep the dashboard static, GitHub Pages-ready and public-safe.
- Do not touch raw GSC exports. Raw CSV files stay local and ignored by Git.

## Global Backlog

- Replace or soften the top "Portfolio in progress" strip so the first signal does not feel unfinished.
- Make the first screen more action-oriented: clearer profile claim, recruiter-friendly CTA, contact path.
- Separate project hierarchy more clearly: core SEO case studies, technical/data workflows and broader marketing experience.
- Avoid giving all projects the same visual weight if some are supporting or secondary.
- Add a compact `Role / Scope / Tools / Output / Evidence` pattern to stronger case studies.
- Keep deep dives for strong projects, but reduce or hide deep dives for lighter social/creator projects.
- Keep jump links for long cases with data, demos or URL inventory; remove or simplify them for lighter cases.
- Watch reveal-on-scroll behavior: full-page screenshots can show unrevealed sections as faded, even though normal scrolling works.

## Hero And First Screen

- Replace the progress strip with a confident status/contact line, for example: `SEO & Digital Marketing Portfolio - Updated July 2026 - Contact`.
- Add stronger first-screen CTAs:
  - `View SEO case studies`
  - `Open CV`
  - optional direct contact / LinkedIn link if available.
- Shorten the hero paragraph so it can be scanned in 8-10 seconds.
- Make the positioning sharper: SEO + WordPress + Search/data analysis + AI-assisted workflows + content/social background.
- Consider adding 3 small proof chips in the hero:
  - `150 SEO pages contributed`
  - `GSC/data reporting`
  - `WordPress-ready HTML modules`
- Consider adding a small "What I do" strip before project cards:
  - `SEO content systems`
  - `Data-backed analysis`
  - `Automation + QA`
- On mobile, reduce first-screen height or add a `View projects` button so the portfolio content is easier to reach.

## Project Hierarchy

- Consider grouping projects instead of showing all 11 as one equal grid:
  - Core SEO case studies: Review Pages, Offer Sheets, Off-page Articles, NRJ2 Analysis, Provider Ranking.
  - Technical/data workflows: Social Scraping, Branch Map, Review Summary.
  - Broader marketing experience: Ladywoods / BGSA, Creator Work, Change Residence.
- Consider moving Branch Map after NRJ2 Analysis because it is a personal prototype connected to the NRJ2 diagnosis.
- Consider keeping Change-of-Residence last and visually lighter.
- Consider presenting Review Pages, Ranking and Review Summary as one review ecosystem rather than disconnected cards.
